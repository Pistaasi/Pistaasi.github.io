Mitä teit / minkä komennon annoit

Katsoin aluksi selaimen (chrome) developer näkymästä Network välilehden omalta sivultani (hostattuna herokussa) ja sitten YouTuben sivuilta.


Mitä tapahtui / mitä tuli vastaukseksi

Selaimen ikkuna omalla sivullani näytti kivassa järjestyksessä mitä sivun osia on ladattu (Omalla sivulla html dokumentti, css tiedosto, favicon ikoni ja login sivu).
Chromen Network näkymä näyttää myös tietoja kuten statuksen, tyypin, Initiator (kuka yhteyttä on kutsunut olettaisin, esimerkiksi login sivua on kutsunut favivon.ico), tiedoston koon, ajan (kauanko kesti) ja Waterfall näkymän, joka kertoo lisätietoja hausta.
Esim. css tiedostoni ilmoittaa milloin lataus on asetettu jonoon, milloin lataus on aloitettu ,kauanko se kesti eri osissa.
Waterfallissa on myös graafi, johon on merkattu vihreällä latauksen osiot (Waiting for server response, Content Download).
Omalla sivullani näkyvät yhteydet olivat hieman outoja
- kutsuu favicon.icoa, jota ei ole olemassa
- favicon.ico kutsuu login sivua, koska en ole merkannut sitä hyväksytyksi sivuksi kirjautumatta (joka tarkoittaa redirectiä login sivulle)
Kirjautumisen ja pois kirjautumisen jälkeen yhteydet katosivat. Kirjautuneena näkyi, ettei favicon.icoon voitu ottaa yhteyttä, jonka jälkeen se ei sitä yrittänytkään. En tiedä onko herokun juttua vai ei katsomatta syvemmin.

Mitä teit / minkä komennon annoit

Wiresharkilla testatessa valitsin WLAN yhteyden ja painoin starttia.

Mitä tapahtui / mitä tuli vastaukseksi

WLAN yhteys seuraa oman tietokoneeni yhteyksiä WLAN yhteyksien kanssa (esim juurikin google.com, joka minulla on auki)
- Näytölle ilmestyy paljon IP osoitteita, joista ilmoitetaan aika, source (osoite), destination, protocol (protokolla, esim TCP näkyy välillä)
- Esimerkiksi Standard query history.google.com (DNS protokolla, olisiko kysynyt googlen historian DNS?)
- Näkyy myös paljon Receiver reportteja, RTCP protokollasta. Taitavat olla vastaanottoraportteja onnistuneesta lähetyksestä tiedonsiirrossa).
